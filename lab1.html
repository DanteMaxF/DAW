Conceptos

URL

URL por sus siglas significa "Uniform Resource Locator", es decir, es una dirección para acceder a algún recurso en el Internet. [3]

MIME

MIME por sus siglas significa "Multipurpose Internet Mail Extensions", el cual es una regla estandarizada del internet para indicar la naturalieza y el formato de un documento. Algunos navegadores utilizan este recurso para determinar como procesar un documento. [4]

HTTP

El protocolo HTTP ofrece retroalimentación mediante distintos códigos para que los usuarios conozcan el estado del servidor. Todos los codigos de tipo 1XX tienen el proposito de informar sobre procesos que se están llevando a cabo. Los códigos de tipo 2XX se despliegan cuando el servidor ofrece respuestas satisfactorias. Cuando la respuesta es un código de tipo 3XX se usa para informar sobre redirecciones. Los errores 4XX informan de errores provenientes del cliente, mientras que los errores 5XX son errores del servidor. [2]

XML

Las siglas para XML significan "eXtensible Markup Language". Éste lenguaje tiene cierto parecido al HTML, sólo que éste es auto-descriptible y fue diseñado para transportar o almacenar información[6]

XHTML

Éste es un lenguaje muy parecido a lo que es el HTML, pero a diferencia del HTML, el XHTML es un lenguaje más estricto y es soportado por la mayoría de los navegadores[7]

HTML5

El HTML5 es la versión estandarizada más actual del lenguaje HTML. Gracias a los nuevos elementos, atributos y comportamientos que ofrece este lenguaje permite darle a los proyectos más alcance con una mayor facilidad.[8]

JS (Javascript)

Javascript es un lenguaje interpretado, orientado a objetos. Se le conoce también como el lenguaje de script para páginas web. Es dinámico, soporta estilos de programación funcional e imperativa. [9]

PHP

PHP o "Hypertext Preprocessor" es un lenguaje open source que los desarrolladores web utilizan para darle más alcance a sus proyectos, ya que éste puede ser incrustado directamente al lenguaje de HTML.[10]

JSP

La tecnología JSP (JavaServer Pages) es utilizada para desarrollar contenido dinámico web de manera más rápida. Ésta tecnología permite crear aplicaciones que se puedan ejecutar desde distintas plataformas, pues Java es un lenguaje multiplataforma.[11]

Ajax

Ajax es una tecnología para realizar una conexión entre un servidor y un navegador usando Javascript. Es una tecnología muy utilizada, pues permite al desarrollador actualizar la página web sin neceseidad de cargar nuevamente y enviar información al servidor web "por atrás".[12]

Preguntas

  1. ¿Cuál es la diferencia entre Internet y la World Wide Web?
    • El internet es una red conformada por otras redes en donde miles de millones de computadoras se encuentran conectadas mediante una variedad de distintos protocolos. Mientras que la World Wide Web es tan sólo uno de los medios por los cuales te puedes conectar al internet, subir y obtener información mediante protocolos HTTP [5]

  2. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
    • GET: Sirve para recibir información del servidor. Únicamente se usa para leer la información.
    • HEAD: Muy parecido a GET, pero no la respuesta viene sin el "body".
    • POST: Utilizado para enviar información al servidor. Igual se puede utilizar GET pero por cuestiones de seguridad es recomendado usar POST.
    • PUT: Parecido a POST, pero PUT es más usado para actualizar contenido de los servidores.
    • PATCH: Parecido a POST, pero PATCH es utilizado para realizar actualizaciones parciales de información de los servidores.
    • DELETE: Utilizado para borrar información del servidor. [1]

Investigación

Ciclo de vida de los sistemas de información

[16]

El ciclo de vida de un sistema de información es un concepto que define las etapas de organización de un proyecto de desarrollo de un sistema, desde su nacimiento hasta que el mismo se vuelve obsoleto y nace otro sistema para sustituirlo. Existen muchos enfoques para definir un ciclo de vida, pero el más aceptado a la fecha es el de cascada, el cual cada etapa es indispensable para avanzar a la siguiente:

Investigación preliminar

Antes de comenzar cualquier proyecto, es necesario realizar una breve investigación sobre la organización para la que se realizará el sistema; en esta etapa se define el problema que se tiene y las posibles soluciones al mismo para empezar a establecer el alcance del proyecto. Para esta etapa es importante mantener una fuerte comunicación con la organización.

Análisis detallado de la situación existente

En esta etapa se comprende el problema de manera más profunda. Se analiza la manera en que la organización realiza la gestión de la información. Para ésta etapa es necesario realizar entrevistas y análisis de varios documentos. Con esa información ya se puede empezar a diseñar a un alto nivel algunas posibles soluciones para la siguiente etapa.

Diseño del nuevo sistema

Durante esta etapa, se empezarán a diseñar los requisitos funcionales fundamentales que deberá de tener el sistema, los cuales estarán basados en la solución de las problemáticas definidas en la etapa anterior. Es posible que también se diseñen nuevos elementos que no son realmente necesarios para el sistema, sin embargo, le pueden dar valor al proyecto. Para identificar la importancia de cada requisito funcional del sistema, es necesario analizarlos y darles un nivel de prioridad.

Desarrollo y documentación del nuevo sistema

Una vez definido el alcance del proyecto, se procede a analizar la metodología en la cual se desarrollará el sistema: Si se va adquirir un proyecto genérico y se adaptará a las necesidades del cliente o si se desarrollará un software nuevo. Para la segunda opción se tendrán que definir las técnicas y se procederá a realizar la elaboración de programas con los lenguajes establecidos, a eso se le conoce como "diseño físico del sistema" y se basa en la documentación de las etapas pasadas.

Implantación del sistema de información y conocimientos

Aquí es cuando el proyecto ya se encuentra casi terminado, y se introduce el mismo en el ambiente práctico. Para ésta etapa será necesario realizar una fase experimental en donde se realizan pruebas del sistema y en su caso, se ajustarán los detalles que requieran mejora para ofrecer un sistema sólido y de calidad.

Mantenimiento del sistema

El objetivo de esta etapa consiste en adecuar el sistema a los cambios del entorno en los que se puede encontrar, de ésta manera se previene que el sistema se vuelva obsoleto con el paso del tiempo.
